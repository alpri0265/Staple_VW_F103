# Staple_VW_F103 — підключення та розпіновка

Цільова плата: **STM32F103C8T6 (BluePill)**, логіка **3.3V**.

## Розпіновка (MCU → сигнал → модуль)

| STM32F103 pin | Сигнал у коді | Призначення | Примітки |
|---|---|---|---|
| PA0 | `STOP_BTN` (`STOP_Pin`) | STOP кнопка (EXTI0) | Активний **LOW** (кнопка на GND) |
| PA1 | `ENC_CLK` (`CLK_Pin`) | Енкодер KY-040 CLK (EXTI1) | Фронт по EXTI, бажано підтяжка |
| PA2 | `ENC_DT` (`DT_Pin`) | Енкодер KY-040 DT | Зчитується в ISR по CLK |
| PA3 | `ENC_SW` (`SW_Pin`) | Кнопка енкодера | Активний **LOW** (на GND), debounce 1ms tick |
| PB0 | `HX_DAT` (`DAT_Pin`) | HX711 DT/DOUT | Вхід, готовність: LOW |
| PB1 | `HX_CLK` (`HX_CLK_Pin`) | HX711 SCK/CLK | Вихід (bit-bang) |
| PB6 | I2C1_SCL | LCD2004 через PCF8574 (I2C) | Потрібні pull-up резистори |
| PB7 | I2C1_SDA | LCD2004 через PCF8574 (I2C) | Потрібні pull-up резистори |
| PB10 | `LIMIT_TOP` (`TOP_Pin`) | Верхній кінцевик | Активний **LOW** (на GND) |
| PB11 | `LIMIT_BOT` (`BOT_Pin`) | Нижній кінцевик | Активний **LOW** (на GND) |
| PB12 | `JOY_UP` (`UP_Pin`) | Джойстик UP | Активний **LOW** (на GND), debounce 1ms tick |
| PB13 | `JOY_DOWN` (`DOWN_Pin`) | Джойстик DOWN | Активний **LOW** (на GND), debounce 1ms tick |
| PA8 | `STEP` (`STEP_Pin`) | STEP драйвера двигуна | Генерується софтом з 1ms tick |
| PA9 | `DIR` (`DIR_Pin`) | DIR драйвера двигуна | Напрям |
| PA10 | `ENABLE` (`ENABLE_Pin`) | ENABLE драйвера двигуна | **Active-low enable**: `0=enable`, `1=disable` |

Джерело: визначення пінів у [main.h](file:///home/alpri/Documents/PlatformIO/Projects/Staple_VW_F103/Core/Inc/main.h).

## Підключення модулів

### Драйвер крокового двигуна (STEP/DIR/ENABLE)
- `STEP` → PA8
- `DIR` → PA9
- `ENABLE` → PA10 (**0=увімкнено**, **1=вимкнено**)
- GND драйвера ↔ GND BluePill (спільна земля обов’язкова)
- Живлення двигуна/драйвера — окремо (за специфікацією драйвера)

### HX711 (ваговий датчик)
- `DT/DOUT` → PB0
- `SCK/CLK` → PB1
- VCC: рекомендовано живити **3.3V**, щоб рівні логіки гарантовано були 3.3V
- GND ↔ GND (спільна земля)

### LCD2004 + PCF8574 (I2C)
- `SCL` → PB6 (I2C1_SCL)
- `SDA` → PB7 (I2C1_SDA)
- Адреса за замовчуванням у коді: `0x27` (передається як `0x27<<1`)
- Потрібні підтяжки I2C (типово **4.7k** до напруги шини)
  - Якщо LCD/PCF8574 живиться від **5V**, бажано робити **level shifting** або підтяжки до **3.3V**, щоб не подати 5V на PB6/PB7.

### Енкодер KY-040
- `CLK` → PA1 (EXTI1)
- `DT` → PA2
- `SW` → PA3
- VCC: залежить від модуля. Багато плат KY-040 мають підтяжки до VCC. Рекомендовано **3.3V**, щоб рівні були сумісні.
- GND ↔ GND

### Кнопки/кінцевики/джойстик (активний LOW)
- Усі входи в проєкті очікують активний стан як **LOW** (контакт на GND).
- STOP: PA0
- TOP: PB10
- BOT: PB11
- JOY_UP: PB12
- JOY_DOWN: PB13

## Примітки по логіці та безпеці
- STOP працює через EXTI і ставить аварійний стан; вихід з ERROR — підтвердженням (подвійне натискання).
- `ENABLE` драйвера вимикається при аварії (ставиться HIGH).
- I2C LCD має збільшений timeout і спробу відновлення шини при помилці.
